# LeetCode Problems
- This is my personal practice repository for LeetCode problems, which contain the solution-codes written in **Python(3)**, **C**, and **C++**. 

- The solutions are not necessarily optimised for computational efficiency, since I am mostly exploring different algorithms and approaches for each problem. 

- I will also ocassionally modify the problem constraints based on what I might find to be interesting and worthwhile exploring. I will also attempt to solve these modified problems solely based on approaches that might pique my interest and may not necessarily be the optimal way to solve them. 

- Though I have been working with C / C++ for a long time now, my skills are still intermediate at best, and I frequently strucgle with the Object-Oriented-Programming (OOP) aspect for these languages. Though Python is also an OOP language, it is still streets-ahead, when compared to C / C++ (for an intermediate programmer like me). So, for each problem, at first I will attempt Python solutions, and then C / C++.

- So, the C / C++ solutions may not be present for each problem and will be worked on as I progress with my coding skills. 

## Structure
```text
leetcode-problems/
├── README.md
├── 0001-first_problem_id/
    ├── README.md
    ├── 00-main_problem/
    │   ├── problem-description.md
    │   ├── Python-code/
    │   │   ├── README.md
    │   │   ├── 01-first_algorithm.py
    │   │   ├── 02-second_algorithm.py
    │   │   ├── ...
    │   │   ├── NN-nth_algorithm.py
    │   │   └── solver-script.sh
    │   ├── C-code/...
    │   └── C++-code/...
    ├── 01-first_modified_problem/...
    ├── 02-second_modified_problem/...
    ├── ...
    └── NN-nth_modified_problem/...
├── 0002-second_problem_id/...
├── ...
└── NNNN-nth_problem_id/..
```

- Each problem gets its own folder, which will be numberred by the LeetCode problem ID (`0001-two-sum`, `0002-add-two-number`, ...). 

- Inside each problem-folder, there will be at least one sub-folder, which will directly address the LeetCode problem. 

- If I intend to modify the problem description, then there will be additional sub-folder(s), uniquely numbered by their respective problem ID(s).

- Inside each sub-folder, there will be three sub-sub-folder, each dedicated to one of three main programming lamguages --- Python, C, and C++.

- Inside each sub-sub-folder, there be at least one code solving the respective problem. If I am able to figure out new algorithm(s) or approach(es), then there will additional code(s) present in this sub-sub-folder. 

- Furthermore, along with the solution-code, there will be a solver-script that will solve the code for either leetcode cases or my own custom cases. 

- If there are more than one solution-codes present, due to presence of two or more algorithms, then there might also be an additional script that will run a comparision, checking which solution approach is optimal --- in terms of memory usage, computation time, complexity, etc.
